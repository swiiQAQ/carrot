git命令

Veröffentlicht am 2017-08-27

分支


Git branch 查看当前分支
Git branch -a 查看远程分支和当前分支
Git branch test 建立新分支


Git branch -d branchName 删除本地某分支
Git branch -r -d origin/branchName 删除远程分支
Git push origin: branchName 删除远程分支

查看分支之间的关系和commit日志

git log –graph –decorate –oneline –simplify-by-decoration –all

说明：
–decorate 标记会让git log显示每个commit的引用(如:分支、tag等)
–oneline 一行显示
–simplify-by-decoration 只显示被branch或tag引用的commit
–all 表示显示所有的branch，这里也可以选择，比如我指向显示分支ABC的关系，则将–all替换为branchA branchB branchC

查看更改


Git show 查看某次更改的内容，不填写提交ID的话，显示最近一次更改的内容，id通过git log查询

撤销更改

 撤销本地修改
跟踪的文件撤销修改
 git checkout .
未跟踪的文件撤销修改
git clean -n        查看将要清除的文件
git clean -df         删除未跟踪的文件

撤销暂存,暂存的文件取消暂存变成已修改未暂存

Git reset HEAD b.txt

更改commit提交信息

Git commit –amend进入编辑器更改提交信息
或者
Git commit –amend -m

将提交信息由change b 更改为change b.txt,
将提交信息由change b.txt更改为rechange b.txt

撤销提交(commit)文件

Git reset –hard

Rechange b.txt的更改全部丢弃，本地也没有修改的文件

Git reset

reset后修改的文件仍然保留在本地磁盘

在撤销commit后再恢复

reset后又还是想commit刚刚撤销的文件
Git reflog
Git cherry-pick
查看HEAD改变的历史记录，它涉及的只是 HEAD 的改变。在你切换分支、用 git commit 进行提交、以及用 git reset 撤销 commit 时，HEAD 会改变，但当你用 git checkout – 撤销时，HEAD 并不会改变 — 如前所述，这些修改从来没有被提交过，因此 reflog 也无法帮助我们恢复它们。

撤销merge

道理同撤销commit一样
Git reset –hard
如果merge出现冲突的话
Git merge –abort

撤销已经push的内容

Git revert HEAD 撤销前一次push
Git revert HEAD^ 撤销前前一次push
Git revert
但是并不是之前提交的东西在分支上没有，还是会有记录的，另外假如后悔revert后，可以再revert一遍…
